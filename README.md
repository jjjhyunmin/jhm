# 📦 장비 관리 시스템 (Asset Management System)

회사 내부 장비(비품)를 효율적으로 **등록 · 대여 · 반납 · 수리**까지 관리할 수 있도록 제작된 웹 기반 장비 관리 시스템입니다.  
관리자와 사용자의 역할을 명확히 분리하여 장비 흐름을 투명하고 안전하게 관리할 수 있도록 설계되었습니다.

---

## ✨ 주요 기능

### ✔ 1. 장비 등록 (관리자)
관리자가 새로운 장비를 시스템에 등록합니다.  
등록 시 작성 항목:

- 장비명  
- 장비 수량  
- 장비 가격  
- 비고(설명)

---

### ✔ 2. 장비 대여 신청서 작성 (사용자)
사용자는 장비 대여를 위해 **대여 신청서를 작성하여 관리자에게 전달**합니다.  
작성 항목:

- 대여 수량  
- 대여 기간  
- 이름  
- 소속  
- 직급  
- 비밀번호 *(반납 시 인증용)*

비밀번호는 오직 본인만 반납할 수 있도록 하기 위해 사용됩니다.

---

### ✔ 3. 관리자 승인 / 거절 처리
관리자는 사용자로부터 받은 대여 요청을 검토 후:

- **승인**  
- **거절 (거절 사유 입력)**  

둘 중 하나를 선택할 수 있습니다.

승인 시 해당 장비는 대여 상태로 전환되며, 잔여 수량이 감소합니다.

---

### ✔ 4. 장비 반납 (사용자)
사용자는 장비 사용을 마치면 반납을 진행합니다.

- 반납 시 **대여 신청서에 입력한 비밀번호를 다시 입력해야 반납 가능**
- 장비가 파손된 경우 **파손 사유 입력 필수**

비밀번호 인증을 통해 타인이 임의로 반납하는 행위를 방지합니다.

---

### ✔ 5. 장비 파손 및 수리 관리 (관리자)
사용자가 반납한 장비가 파손된 경우:

- 관리자가 **수리 요청 등록**
- 실제 수리 완료 후 해당 장비의 **대여 가능 수량 증가**

장비의 상태와 재고 수량을 정확하게 유지할 수 있습니다.

---

### ✔ 6. 비밀번호 인증 기반 보안 기능
대여 신청 시 입력하는 사용자 비밀번호는 다음을 방지하기 위해 사용됩니다:

- 타인의 임의 반납 처리  
- 본인이 아닌 사람이 대여/반납을 악용하는 문제  
- 대여 기록 조작 및 허위 반납

---

## 🛠 사용 기술 (예시)

- PHP  
- MySQL (또는 MariaDB)  
- Apache (XAMPP 환경)  
- HTML, CSS, JavaScript  
- GitHub for Version Control  

---

## 📌 시스템 흐름 요약

1. 관리자: 장비 등록  
2. 사용자: 대여 신청  
3. 관리자: 승인 / 거절  
4. 사용자: 비밀번호 입력 후 반납  
5. 사용자: 파손 사유 입력 가능  
6. 관리자: 수리 요청 → 완료 후 수량 증가
